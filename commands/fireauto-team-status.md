# /fireauto-team-status - 팀 에이전트 상태 확인

실행중인 팀 에이전트의 현재 상태를 확인하고 보고합니다.

## 확인 절차

### 1. 활성 플랜 탐색

`.claude/plans/` 디렉토리에서 현재 활성 상태인 플랜 파일을 검색합니다.

```
Glob: .claude/plans/*.md
```

각 플랜 파일을 읽어서 `상태: 진행중`인 플랜을 식별합니다.

### 2. 워크스트림 상태 파싱

활성 플랜 파일에서 각 워크스트림의 상태를 추출합니다:

- 워크스트림 이름
- 현재 상태 (대기 / 진행중 / 검토필요 / 완료 / 실패)
- 담당 에이전트

### 3. Worktree 상태 확인

git worktree가 사용 중인 경우 각 worktree의 상태를 확인합니다:

```bash
git worktree list
```

각 worktree 브랜치의 최근 커밋과 변경사항을 확인합니다:

```bash
# 각 worktree에 대해
git -C .worktrees/ws-{N}-{name} log --oneline -5
git -C .worktrees/ws-{N}-{name} diff --stat
```

### 4. 상태 리포트 출력

다음 형식으로 상태를 보고합니다:

```
============================================
  fireauto-team 상태 리포트
============================================

플랜: {플랜 이름}
생성: {생성일시}
전체 진행률: {완료 워크스트림 수}/{전체 워크스트림 수}

--------------------------------------------
워크스트림 상태
--------------------------------------------

  WS-1: {이름}
    상태: {상태 아이콘} {상태}
    브랜치: fireauto/ws-1-{name}
    최근 변경: {파일 수}개 파일, +{추가 라인}/-{삭제 라인}

  WS-2: {이름}
    상태: {상태 아이콘} {상태}
    브랜치: fireauto/ws-2-{name}
    최근 변경: {파일 수}개 파일, +{추가 라인}/-{삭제 라인}

  ...

--------------------------------------------
충돌 감지
--------------------------------------------
  공유 파일 수정 감지: {있음/없음}
  {충돌 파일 목록이 있다면 표시}

============================================
```

### 상태 아이콘 매핑

- 대기: [대기]
- 진행중: [진행]
- 검토필요: [검토]
- 완료: [완료]
- 실패: [실패]

### 5. 충돌 사전 감지

여러 worktree에서 동일한 파일을 수정한 경우를 감지합니다:

```bash
# 각 worktree의 변경 파일 목록 추출 후 교집합 확인
git -C .worktrees/ws-{N}-{name} diff --name-only main
```

교집합이 발견되면 충돌 가능성을 경고합니다.

### 6. 추가 조치 안내

상태에 따라 사용자에게 다음 조치를 안내합니다:

- **모든 워크스트림 완료**: "모든 작업이 완료되었습니다. 병합을 진행하시겠습니까?"
- **일부 실패**: "실패한 워크스트림을 재시작하거나, 수동으로 처리하시겠습니까?"
- **검토 필요**: "다음 워크스트림이 검토를 요청합니다: {목록}"
- **충돌 감지**: "다음 파일에서 충돌이 예상됩니다: {목록}. 병합 전 수동 확인을 권장합니다."
